# Contributing

1. Fork the repo and create a branch from `main`.
2. Run `npm install` then `npm run dev` to confirm everything works locally.
3. Make your changes. Run `npm run lint` and `npm run build` before opening a PR -- both must pass with zero errors.
4. Keep PRs focused. One feature or fix per PR.
5. Open a pull request against `main` with a short description of what changed and why.

Code style is enforced by Prettier and ESLint. Run `npm run format` to auto-format before committing.
